# Code Execution Feature

## Overview

Gemini Code Execution provides a remote Python environment for executing code generated by the AI. The code runs in a sandboxed cloud environment with access to common data science libraries.

## How It Works

The AI generates Python code that is executed in a remote environment:

```python
from google.genai import Client, types

client = Client(api_key=os.getenv("GEMINI_API_KEY"))

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=[uploaded_file, prompt],
    config=types.GenerateContentConfig(
        tools=[types.Tool(code_execution=types.ToolCodeExecution())]
    )
)
```

## Available Libraries

Based on testing, the following Python libraries are available:

### Data Manipulation
- **pandas** - DataFrame operations
- **numpy** - Numerical computing
- **os** - File system operations

### Statistics
- **scipy.stats** - Statistical tests
- **statsmodels** - Statistical modeling

### Visualization
- **matplotlib** - Basic plotting
- **seaborn** - Statistical visualization

## Execution Flow

1. **Code Generation**: AI generates Python code based on the prompt
2. **Code Execution**: Generated code runs in the remote environment
3. **Result Capture**: Output (stdout/stderr) is captured
4. **Response**: Results and any generated files are returned

## Raw Output Structure

The response contains multiple parts:

### Executable Code
```python
ExecutableCode(
    code="""import pandas as pd
import numpy as np
# ... more code
""",
    language='PYTHON'
)
```

### Code Execution Result
```
Outcome: Outcome.OUTCOME_OK
Output:
DataFrame Info:
<class 'pandas.core.frame.DataFrame'>
RangeIndex: 150 entries, 0 to 149
...
```

### Inline Data
Binary data (images, CSVs) generated during execution:
```
Blob(
    data=b'\x89PNG\r\n\x1a\n...',
    mime_type='image/png'
)
```

## Error Handling

The code execution handles errors gracefully:

### Execution Errors
```
Outcome: Outcome.OUTCOME_FAILED
Output:
Traceback (most recent call last):
  File "<string>", line 16, in <module>
    ...
PerfectSeparationError: Perfect separation detected, results not available
```

The AI can detect errors and attempt recovery:
- When logistic regression failed due to perfect separation, the AI skipped it and continued with visualization

## Supported Models

The following models support code execution:
- `gemini-2.5-flash` (tested)
- `gemini-2.5-pro`
- `gemini-2.0-flash`

Note: Some models like `gemini-2.0-flash-exp` do not support code execution.

## Limitations

1. **Remote Execution**: Code runs in Google's cloud, not locally
2. **Execution Time**: Long-running code may timeout
3. **Memory Limits**: Large datasets may exceed memory constraints
4. **Network Access**: Limited or no external network access during execution
5. **File Persistence**: Generated files are temporary

## Best Practices

1. **Incremental Complexity**: Start with simple operations, build up complexity
2. **Error Recovery**: Design prompts to handle expected errors gracefully
3. **Output Capture**: Always capture and check execution results
4. **File Size**: Keep generated files reasonably sized
5. **Timeout Awareness**: Be aware of execution time limits
